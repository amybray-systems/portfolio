Key Lessons Learned

1) Flask over Django for speed. I needed to ship pages, not frameworks. Minimal ceremony wins during proof-of-value.

2) Roles are product, not just permissions. Contractors, managers, and admins think in different workflows. Designing by role tasks simplified UI and reduced conditionals.

3) Dark mode isn‚Äôt ‚Äújust a theme.‚Äù A 3-day rabbit hole taught me to centralize CSS variables (--bg, --text, spacing) and reuse components.

4) ‚ÄúMake it work‚Äù first, then harden it. JSON let me iterate fast. Now that flows are stable, migrating to a DB is straightforward and justified.

5) AI shines when it‚Äôs narrow and contextual. Aunikah handles FAQs, suggests next steps, and logs escalations. Using two LLMs for different strengths (ChatGPT for speed, Claude for polish) noticeably sped up shipping.

Biggest Challenges (and fixes)

Challenge 1: UI Drift

Problem: Forms looked fine in one view and broken in another (light mode unreadable, odd spacing).

Solution: Centralized form styles & typography, introduced design tokens with CSS custom properties, and tested with real content.

Takeaway: Design tokens first. Test with real data, not lorem ipsum.

Challenge 2: Role Logic Sprawl

Problem: Templates filled with if admin/manager/contractor conditionals.

Solution: Route-level guards + role-scoped partials. Each role gets a dedicated dashboard with only the actions they need.

Takeaway: Treat roles like distinct products.

Challenge 3: Files Work‚Ä¶ Until They Don‚Äôt

Problem: Concurrent writes are risky and analytics (e.g., kudos by tag/week) can get slow.

Solution: Abstracted file access behind helpers that can be swapped for SQLite/Postgres.

Takeaway: Prototype with files, architect for a DB.
