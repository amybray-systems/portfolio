Project Overview

VMSFlex Pay was born from a real pain point I experienced in previous roles: spending hours each pay period manually processing timesheets from multiple Vendor Management Systems, each with completely different CSV formats, and calculating overtime by hand based on complex location-specific rules.

The solution uses intelligent format detection to automatically recognize various CSV structures, handle different date and time formats, and apply the correct overtime rules based on employee location (California daily OT, Federal weekly OT, etc.). What used to take 2-3 hours now takes minutes.

Smart Detection Engine

The core innovation is the SmartCSVParser class that uses pattern matching to flexibly identify columns regardless of naming conventions. It can detect:

  • Employee ID variations (Employee ID, EmpID, Worker ID, Staff ID, etc.)
  • Multiple date formats (MM/DD/YYYY, YYYY-MM-DD, DD-MM-YYYY)
  • Different hour formats (8.5, 8:30, 8h 30m)
  • Pre-calculated vs. total hours format

Overtime Calculation Logic

The system routes calculations based on whether the VMS pre-calculated overtime or provided only total hours:

  • Pre-calculated format: Uses VMS-provided Regular/OT/DT breakdown
  • Total hours format: Applies location-specific rules to calculate OT/DT
  • California rules: Daily OT (>8hrs), Daily DT (>12hrs), Weekly OT (>40hrs), 7th day rules
  • Federal/NY rules: Weekly OT threshold (>40hrs)

The Problem It Solves

In previous roles, payroll administrators had to:

  • Manually pull hours from multiple VMS accounts
  • Deal with inconsistent CSV formats across different systems
  • Calculate overtime by hand or use error-prone Excel formulas
  • Apply different overtime rules based on employee location
  • Handle scenarios where some VMS exports show total hours only, while others pre-calculate regular/OT/DT breakdowns

This manual process was time-consuming (2-3 hours per pay period), error-prone, and didn't scale as the workforce grew.
